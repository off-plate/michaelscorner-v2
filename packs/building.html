PACK/03 · Building

Building software with AI

8 prompts · updated 07/2026

Go from a rough idea to working software, even if you do not code. Plan small, brief the AI properly, and get unstuck when it breaks. It will break.

What this pack does, honestly:

It plans, briefs, and unblocks the build. It does not test your product with real users. You do.

01

Rough idea into a build spec

Use this when you have an idea in your head and need a plan before you let any tool write code.

 → 02

Plan the smallest version worth shipping

Use this when your idea keeps growing and you need to ship something small first.

 → 03

Brief an AI coding tool

Use this when the spec is ready and you want the first message to Claude Code, Cursor, or a similar tool to start the build right.

 → 04

Explain and fix this error

Use this when you hit an error and you are not sure what the code even does.

 → 05

Get out of the fix loop

Use this when the AI keeps confidently fixing the same bug and the bug keeps not being fixed.

 → 06

Tell me what this code does

Use this when you are about to use code you did not write and want to know what it really does first.

 → 07

Pick tools without the hype

Use this when every tool claims to be the answer and you just need one that fits your case.

 → 08

Test it before anyone sees it

Use this when the build looks done and you want to find the embarrassing bugs before your first user does.
